
mocha本身不带断言库，所以必须先引入断言库。
mocha可以使用各种断言库，如chai，并指定使用它的expect断言风格。
eg:
	开发代码(add.js)：
		function add(x, y) {
		  return x + y;
		}

		module.exports = add;
	测试代码：
		var add = require('./add.js');
		var expect = require('chai').expect;

		describe('加法函数的测试', function() {
		  it('1 加 1 应该等于 2', function() {
		    expect(add(1, 1)).to.be.equal(2);
		  });
		});
	配置：
		"scripts": {
		  "test": "mocha *.test.js"
		}
	启动测试:
		npm run test
说明：
	1. 测试脚本里面应该包括一个或多个describe块，每个describe块应该包括一个或多个it块。
	2. describe块称为"测试套件"（test suite），表示一组相关的测试。它是一个函数，第一个参数是测试套件的名称（"加法函数的测试"），第二个参数是一个实际执行的函数。
	3. it块称为"测试用例"（test case），表示一个单独的测试，是测试的最小单位。它也是一个函数，第一个参数是测试用例的名称（"1 加 1 应该等于2"），第二个参数是一个实际执行的函数。
